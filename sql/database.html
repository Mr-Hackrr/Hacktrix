Data Definition Language (DDL)

Overview of DDL

Data Definition Language (DDL) is a subset of SQL used to define and manage database structures. DDL commands are used to create, modify, and delete database objects such as tables, indexes, and views. The primary DDL commands include CREATE, ALTER, DROP, and TRUNCATE.

CREATE TABLE

The CREATE TABLE statement is used to create a new table in the database.

CREATE TABLE table_name (
        column1 datatype constraints,
        column2 datatype constraints,
        ...
    );

Examples:

Creating a simple table:

CREATE TABLE employees (
        employee_id INT PRIMARY KEY,
        first_name VARCHAR(50),
        last_name VARCHAR(50),
        email VARCHAR(100)
    );

This creates an employees table with columns for employee ID, first name, last name, and email.

Creating a table with a foreign key:

CREATE TABLE orders (
        order_id INT PRIMARY KEY,
        order_date DATE,
        customer_id INT,
        FOREIGN KEY (customer_id) REFERENCES customers(customer_id)
    );

This creates an orders table with a foreign key referencing the customers table.

Creating a table with default values:

CREATE TABLE products (
        product_id INT PRIMARY KEY,
        product_name VARCHAR(100),
        price DECIMAL(10, 2) DEFAULT 0.00
    );

This creates a products table with a default price of 0.00.

ALTER TABLE

The ALTER TABLE statement is used to modify an existing table structure.

ALTER TABLE table_name
    ADD column_name datatype constraints;
    
    ALTER TABLE table_name
    DROP COLUMN column_name;
    
    ALTER TABLE table_name
    MODIFY COLUMN column_name datatype constraints;

Examples:

Adding a new column:

ALTER TABLE employees
    ADD phone_number VARCHAR(15);

This adds a phone_number column to the employees table.

Dropping a column:

ALTER TABLE employees
    DROP COLUMN phone_number;

This removes the phone_number column from the employees table.

Modifying a columnâ€™s data type:

ALTER TABLE employees
    MODIFY COLUMN email VARCHAR(150);

This changes the data type of the email column to VARCHAR(150).

DROP TABLE

The DROP TABLE statement is used to delete an existing table and its data.

DROP TABLE table_name;

Examples:

Dropping a table:

DROP TABLE employees;

This deletes the employees table and all its data.

Dropping a table if it exists:

DROP TABLE IF EXISTS employees;

This deletes the employees table if it exists.

Dropping a table with cascade constraints:

DROP TABLE orders CASCADE;

This deletes the orders table and any dependent objects.

TRUNCATE TABLE

The TRUNCATE TABLE statement is used to remove all rows from a table without deleting the table itself.

TRUNCATE TABLE table_name;

Examples:

Truncating a table:

TRUNCATE TABLE employees;

This removes all rows from the employees table.

Truncating a table with reset identity:

TRUNCATE TABLE employees RESTART IDENTITY;

This removes all rows from the employees table and resets any identity columns.

Truncating a table with cascade:

TRUNCATE TABLE orders CASCADE;

This removes all rows from the orders table and any dependent objects.

Data Manipulation Language (DML)

Overview of DML

Data Manipulation Language (DML) is a subset of SQL used to manage data within database objects. DML commands are used to insert, update, delete, and retrieve data from tables. The primary DML commands include INSERT INTO, UPDATE, DELETE, and SELECT.

INSERT INTO

The INSERT INTO statement is used to add new rows to a table.

INSERT INTO table_name (column1, column2, ...)
    VALUES (value1, value2, ...);

Examples:

Inserting a single row:

INSERT INTO employees (first_name, last_name, email)
    VALUES ('John', 'Doe', 'john.doe@example.com');

This query inserts a new row into the employees table with the specified values.

Inserting multiple rows:

INSERT INTO employees (first_name, last_name, email)
    VALUES 
    ('Jane', 'Smith', 'jane.smith@example.com'),
    ('Robert', 'Brown', 'robert.brown@example.com');

This query inserts two new rows into the employees table.

Inserting data from another table:

INSERT INTO employees_archive (employee_id, first_name, last_name, email)
    SELECT employee_id, first_name, last_name, email
    FROM employees
    WHERE department = 'Sales';

This query inserts rows from the employees table into the employees_archive table for employees in the Sales department.

UPDATE

The UPDATE statement is used to modify existing records in a table.

UPDATE table_name
    SET column1 = value1, column2 = value2, ...
    WHERE condition;

Examples:

Updating a single column:

UPDATE employees
    SET email = 'john.newemail@example.com'
    WHERE first_name = 'John' AND last_name = 'Doe';

This query updates the email address of the employee named John Doe.

Updating multiple columns:

UPDATE employees
    SET first_name = 'Jonathan', email = 'jonathan.doe@example.com'
    WHERE employee_id = 1;

This query updates the first name and email of the employee with ID 1.

Updating based on a subquery:

UPDATE employees
    SET department = 'Marketing'
    WHERE department = (SELECT department_name FROM departments WHERE department_id = 2);

This query updates the department of employees to â€˜Marketingâ€™ where their current department matches the department with ID 2 in the departments table.

DELETE

The DELETE statement is used to remove existing records from a table.

DELETE FROM table_name
    WHERE condition;

Examples:

Deleting a single row:

DELETE FROM employees
    WHERE first_name = 'John' AND last_name = 'Doe';

This query deletes the record of the employee named John Doe from the employees table.

Deleting multiple rows:

DELETE FROM employees
    WHERE department = 'Sales';

This query deletes all employees who work in the Sales department.

Deleting based on a subquery:

DELETE FROM employees
    WHERE department_id = (SELECT department_id FROM departments WHERE department_name = 'HR');

This query deletes employees whose department matches the department named â€˜HRâ€™ in the departments table.

SELECT

The SELECT statement is used to query data from a database. It retrieves data from one or more tables and returns it in the form of a result set.

SELECT column1, column2, ...
    FROM table_name
    WHERE condition;

Examples:

Selecting specific columns:

SELECT first_name, last_name
    FROM employees;

This query retrieves the first_name and last_name columns from the employees table.

Selecting with a WHERE clause:

SELECT first_name, last_name
    FROM employees
    WHERE department = 'Sales';

This query retrieves the first_name and last_name of employees who work in the Sales department.

Selecting with an aggregate function:

SELECT department, COUNT(*) AS employee_count
    FROM employees
    GROUP BY department;

This query retrieves the count of employees in each department.

Data Control Language (DCL)

Overview of DCL

Data Control Language (DCL) is a subset of SQL used to control access to data in a database. DCL commands are used to grant and revoke permissions to users and roles, ensuring that only authorized users can perform certain actions on the database. The primary DCL commands are GRANT and REVOKE.

GRANT

The GRANT statement is used to give users or roles specific privileges on database objects.

GRANT privilege1, privilege2, ...
    ON object
    TO user_or_role;

Examples:

Granting SELECT privilege on a table:

GRANT SELECT
    ON employees
    TO user1;

This grants the SELECT privilege on the employees table to user1.

Granting multiple privileges on a table:

GRANT SELECT, INSERT, UPDATE
    ON employees
    TO user1;

This grants the SELECT, INSERT, and UPDATE privileges on the employees table to user1.

Granting privileges to a role:

GRANT SELECT, INSERT
    ON employees
    TO role1;

This grants the SELECT and INSERT privileges on the employees table to role1.

REVOKE

The REVOKE statement is used to remove previously granted privileges from users or roles.

REVOKE privilege1, privilege2, ...
    ON object
    FROM user_or_role;

Examples:

Revoking SELECT privilege on a table:

REVOKE SELECT
    ON employees
    FROM user1;

This revokes the SELECT privilege on the employees table from user1.

Revoking multiple privileges on a table:

REVOKE SELECT, INSERT, UPDATE
    ON employees
    FROM user1;

This revokes the SELECT, INSERT, and UPDATE privileges on the employees table from user1.

Revoking privileges from a role:

REVOKE SELECT, INSERT
    ON employees
    FROM role1;

This revokes the SELECT and INSERT privileges on the employees table from role1.

Transaction Control Language (TCL)

Overview of TCL

Transaction Control Language (TCL) is a subset of SQL used to manage transactions in a database. Transactions are sequences of operations performed as a single logical unit of work. TCL commands ensure the integrity of data by allowing users to control the commit and rollback of transactions. The primary TCL commands are COMMIT, ROLLBACK, and SAVEPOINT.

COMMIT

The COMMIT statement is used to save all changes made during the current transaction. Once a transaction is committed, the changes are permanent and visible to other users.

COMMIT;

Examples:

Committing a transaction:

INSERT INTO employees (first_name, last_name, email)
    VALUES ('John', 'Doe', 'john.doe@example.com');
    COMMIT;

This inserts a new row into the employees table and commits the transaction, making the change permanent.

Committing multiple operations:

UPDATE employees
    SET email = 'john.newemail@example.com'
    WHERE first_name = 'John' AND last_name = 'Doe';
    
    DELETE FROM employees
    WHERE first_name = 'Jane' AND last_name = 'Smith';
    
    COMMIT;

This updates the email of John Doe and deletes Jane Smithâ€™s record, then commits both changes.

Committing after a series of inserts:

INSERT INTO employees (first_name, last_name, email)
    VALUES ('Alice', 'Johnson', 'alice.johnson@example.com');
    
    INSERT INTO employees (first_name, last_name, email)
    VALUES ('Bob', 'Williams', 'bob.williams@example.com');
    
    COMMIT;

This inserts two new rows into the employees table and commits the transaction.

ROLLBACK

The ROLLBACK statement is used to undo changes made during the current transaction. It reverts the database to the state it was in before the transaction began.

ROLLBACK;

Examples:

Rolling back a transaction:

INSERT INTO employees (first_name, last_name, email)
    VALUES ('John', 'Doe', 'john.doe@example.com');
    ROLLBACK;

This inserts a new row into the employees table but then rolls back the transaction, undoing the insert.

Rolling back multiple operations:

UPDATE employees
    SET email = 'john.newemail@example.com'
    WHERE first_name = 'John' AND last_name = 'Doe';
    
    DELETE FROM employees
    WHERE first_name = 'Jane' AND last_name = 'Smith';
    
    ROLLBACK;

This updates the email of John Doe and deletes Jane Smithâ€™s record, but then rolls back the transaction, undoing both changes.

Rolling back after a series of inserts:

INSERT INTO employees (first_name, last_name, email)
    VALUES ('Alice', 'Johnson', 'alice.johnson@example.com');
    
    INSERT INTO employees (first_name, last_name, email)
    VALUES ('Bob', 'Williams', 'bob.williams@example.com');
    
    ROLLBACK;

This inserts two new rows into the employees table but then rolls back the transaction, undoing both inserts.

SAVEPOINT

The SAVEPOINT statement is used to set a point within a transaction to which you can later roll back. This allows for partial rollbacks within a transaction.

SAVEPOINT savepoint_name;

Examples:

Creating a savepoint:

INSERT INTO employees (first_name, last_name, email)
    VALUES ('John', 'Doe', 'john.doe@example.com');
    
    SAVEPOINT sp1;
    
    INSERT INTO employees (first_name, last_name, email)
    VALUES ('Jane', 'Smith', 'jane.smith@example.com');
    
    ROLLBACK TO sp1;
    COMMIT;

This inserts John Doeâ€™s record, creates a savepoint sp1, inserts Jane Smithâ€™s record, then rolls back to sp1, undoing Jane Smithâ€™s insert but keeping John Doeâ€™s insert.

Using multiple savepoints:

INSERT INTO employees (first_name, last_name, email)
    VALUES ('Alice', 'Johnson', 'alice.johnson@example.com');
    
    SAVEPOINT sp1;
    
    INSERT INTO employees (first_name, last_name, email)
    VALUES ('Bob', 'Williams', 'bob.williams@example.com');
    
    SAVEPOINT sp2;
    
    INSERT INTO employees (first_name, last_name, email)
    VALUES ('Charlie', 'Brown', 'charlie.brown@example.com');
    
    ROLLBACK TO sp2;
    COMMIT;

This inserts Alice Johnsonâ€™s record, creates savepoint sp1, inserts Bob Williamsâ€™ record, creates savepoint sp2, inserts Charlie Brownâ€™s record, then rolls back to sp2, undoing Charlie Brownâ€™s insert but keeping the previous inserts.

Rolling back to a savepoint and continuing:

INSERT INTO employees (first_name, last_name, email)
    VALUES ('David', 'Smith', 'david.smith@example.com');
    
    SAVEPOINT sp1;
    
    INSERT INTO employees (first_name, last_name, email)
    VALUES ('Eve', 'Johnson', 'eve.johnson@example.com');
    
    ROLLBACK TO sp1;
    
    INSERT INTO employees (first_name, last_name, email)
    VALUES ('Frank', 'Miller', 'frank.miller@example.com');
    
    COMMIT;

This inserts David Smithâ€™s record, creates savepoint sp1, inserts Eve Johnsonâ€™s record, rolls back to sp1, then inserts Frank Millerâ€™s record and commits the transaction.
