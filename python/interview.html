Top Python Theory Interview Questions

1. What are the main features of Python?

Python is an interpreted, high-level, dynamically typed language with an emphasis on readability and simplicity. It supports multiple programming paradigms such as object-oriented, procedural, and functional programming. Python also has extensive libraries and is highly portable across platforms.

2. Explain the difference between Python 2 and Python 3.

Python 3 is the current version, actively maintained, while Python 2 was deprecated in 2020. Differences include the 'print' function in Python 3, default Unicode support, changes in integer division, and broader library support in Python 3.

3. How is Python interpreted?

Python code is executed by an interpreter, which translates it line by line into machine code at runtime. The standard interpreter, CPython, compiles code into bytecode, which is then interpreted by the Python virtual machine (PVM).

4. What is PEP 8, and why is it important?

PEP 8 is Python's style guide that provides conventions for writing readable and consistent code. Following PEP 8 ensures that Python code remains clean, organized, and easy to understand for developers.

5. Explain the difference between lists, tuples, and sets.

Lists are mutable, ordered collections that allow duplicate items.

Tuples are immutable, ordered collections that also allow duplicates.

Sets are mutable but unordered collections that do not allow duplicate items and support operations like union and intersection.

6. What are Python's built-in data types?

Some built-in data types in Python include:

  • Numeric: int, float, complex
  • Sequence: list, tuple, range
  • Text: str
  • Mapping: dict
  • Set: set, frozenset
  • Boolean: bool
  • Binary: bytes, bytearray, memoryview
7. How does type conversion work in Python?

Python supports implicit (automatic) and explicit (manual) type conversion. Implicit conversions happen automatically where there’s no risk of data loss, while explicit conversions (e.g., int(), float(), str()) require explicit casting by the programmer.

8. Explain mutable and immutable types in Python.

Mutable types can be changed after creation, while immutable types cannot. For example, lists and dictionaries are mutable, while strings, tuples, and frozensets are immutable.

9. What is the difference between == and is?

== checks if the values of two objects are the same, while is checks if two variables point to the same object in memory (i.e., identity).

10. Explain the concept of lambda functions.

A lambda function is an anonymous, one-line function defined using the lambda keyword. It’s generally used for simple operations and is often used with functions like map, filter, and reduce. Example: lambda x, y: x + y.

11. What are *args and **kwargs?

*args allows a function to accept a variable number of positional arguments as a tuple, while **kwargs allows a function to accept a variable number of keyword arguments as a dictionary. They enable flexible function calling with different numbers of arguments.

12. How does Python handle variable scope and namespaces?

Python uses the LEGB (Local, Enclosing, Global, Built-in) rule to resolve variable scopes. Each scope (local, enclosing, global, and built-in) is a namespace that Python searches in this order to find a variable.

13. What is recursion? Give an example of its use in Python.

Recursion is when a function calls itself to solve smaller subproblems of the main problem. It’s useful for problems like factorial calculation, where the function calls itself with a decreasing number until it reaches a base case.

def factorial(n):
        return 1 if n == 0 else n * factorial(n-1)
14. What is the difference between deep copy and shallow copy?

A shallow copy copies the reference pointers to the objects, so nested objects are shared between the original and the copy. A deep copy copies everything, including nested objects, creating independent copies of each element.

15. What is the difference between Python lists and arrays?

Lists can contain elements of different data types and are more flexible, while arrays (from the array module or numpy) require elements to be of the same data type, making them more memory-efficient and faster for numerical computations.

16. What are Python decorators, and how do they work?

Decorators are functions that modify the behavior of another function or method. They allow you to wrap another function to extend its behavior, typically using the @decorator_name syntax. They are widely used for logging, authentication, and caching.

17. What is the purpose of the __init__ method in Python?

The __init__ method is the constructor in Python classes. It initializes an instance of a class, allowing you to set initial values for object attributes when the object is created.

18. What is a generator in Python?

A generator is a function that yields values one at a time, using the yield keyword, instead of returning them all at once. Generators are memory efficient for large datasets and can be iterated lazily.

19. Explain the concept of list comprehension.

List comprehension is a concise way to create lists based on existing lists. It allows you to write loops and conditional statements within square brackets, making code shorter and often faster.

Example: [x**2 for x in range(10) if x % 2 == 0]

20. What is the purpose of self in Python classes?

self represents the instance of the class and is used to access instance attributes and methods from within the class. It must be the first parameter in any method in a class but does not need to be explicitly passed when calling the method.

21. What is method overloading? Does Python support it?

Method overloading refers to defining multiple methods with the same name but different parameters. Python doesn’t natively support it as other languages do. However, similar functionality can be achieved using default arguments or handling arguments within a single method.

22. What is inheritance in Python?

Inheritance allows a class (child class) to inherit attributes and methods from another class (parent class). Python supports single, multiple, multilevel, hierarchical, and hybrid inheritance.

23. What is polymorphism in Python?

Polymorphism allows methods to do different things based on the object that calls them. For instance, methods in different classes can have the same name but exhibit different behavior.

24. What is encapsulation in Python?

Encapsulation is the concept of bundling data and methods that work on that data within a single unit (class). It also restricts direct access to some attributes, allowing controlled access through methods (getters/setters).

25. Explain the difference between a module and a package in Python.

A module is a single file containing Python code, while a package is a collection of modules organized in directories with an __init__.py file. Packages enable better organization and reuse of code.

26. What is the difference between __str__ and __repr__ in Python?

__str__ is meant to provide a readable, informal string representation of an object (for end-users), while __repr__ provides an official string representation that can be used to recreate the object (for developers).

27. What is monkey patching in Python?

Monkey patching allows you to modify or extend a class or module at runtime. It’s sometimes used to dynamically alter behavior, though it can make debugging harder.

28. What are docstrings in Python?

Docstrings are documentation strings that describe a function, class, or module. They are defined using triple quotes and can be accessed using .__doc__.

29. What is exception handling in Python, and how does it work?

Exception handling allows you to manage runtime errors using try, except, finally, and else blocks. The try block contains the code that may raise an exception, and except handles the exception if it occurs.

30. Explain try, except, finally, and else in Python exception handling.

try: Block where code that might raise an exception is placed.

except: Block that handles the exception if one is raised.

finally: Block that executes regardless of whether an exception occurs, often used for cleanup.

else: Block that runs only if no exception was raised.

31. What are iterators and iterables in Python?

An iterable is an object that can return an iterator, like lists, tuples, or strings. An iterator is an object that allows you to traverse through all the elements of an iterable using __iter__ and __next__.

32. What is the difference between range and xrange?

In Python 2, range generates a list, while xrange generates values lazily as needed. In Python 3, range behaves like xrange, creating a lazy sequence generator, and xrange no longer exists.

33. What is a closure in Python?

A closure is a function defined inside another function that retains access to the outer function’s variables, even after the outer function has completed execution. Closures help in data hiding.

34. What are Python’s memory management mechanisms?

Python uses automatic memory management with a garbage collector to handle unused objects, as well as reference counting. Additionally, the gc module allows for explicit garbage collection control.

35. What is the with statement in Python?

The with statement simplifies exception handling by encapsulating actions within a context (like file handling) and ensuring resources are properly released. It’s often used with open() to manage files.

36. What is a metaclass in Python?

A metaclass is a class for classes. It controls the creation and behavior of classes, allowing developers to modify class attributes and methods at the time of class creation.

37. What is the purpose of the __main__ variable in Python?

__main__ allows you to control the execution of code in a module. When the script is run directly, __name__ == "__main__", but when it’s imported, __name__ will be the module name.

38. What is the purpose of the nonlocal keyword?

The nonlocal keyword allows you to modify a variable in an enclosing (but non-global) scope, often useful in nested functions to update variables from an outer function.

39. What is the difference between shallow copy and deep copy?

A shallow copy copies only the references of nested objects, so changes to nested objects affect both copies. A deep copy creates copies of all nested objects, so changes are isolated.

40. What are map(), filter(), and reduce() functions?

map(): Applies a function to every item in an iterable.

filter(): Filters items in an iterable based on a condition.

reduce(): Reduces an iterable to a single value based on a function (from functools).

41. What is the purpose of __call__ in Python?

The __call__ method allows an instance of a class to be called like a function. This is useful for creating callable objects or mimicking function behavior.

42. What is the difference between staticmethod and classmethod?

staticmethod: A method that does not require an instance or class reference; it acts like a regular function within a class.

classmethod: A method that takes a class (cls) as its first parameter and can access class-level attributes.

43. Explain the concept of duck typing in Python.

Duck typing is a concept where the type or class of an object is less important than its methods and attributes. If an object behaves like the expected type, it is accepted, allowing flexible code.

44. What is the use of the gc module in Python?

The gc module provides an interface to the garbage collection mechanism in Python, allowing developers to manage and debug memory allocation.

45. What are the different types of arguments in Python?

Python has positional arguments, keyword arguments, default arguments, and variable-length arguments (*args for positional and **kwargs for keyword).

46. What is the difference between synchronous and asynchronous programming in Python?

Synchronous programming executes tasks sequentially, waiting for each task to complete before moving to the next. Asynchronous programming allows tasks to run concurrently, using async and await keywords, improving efficiency for I/O-bound operations.

47. What are Python's comprehensions and how do they improve code readability?

Python comprehensions (list, set, and dictionary comprehensions) provide a concise way to create collections. They improve readability by reducing the need for explicit loops and conditional statements.

48. How does Python's context management protocol work?

Python's context management protocol, implemented using __enter__ and __exit__ methods, ensures that resources are properly managed. The with statement is used to wrap the execution of a block of code, ensuring that resources are released when the block is exited.

49. What are Python's data classes and how do they simplify class creation?

Data classes, introduced in Python 3.7, provide a decorator and functions for automatically adding special methods to user-defined classes. They simplify class creation by generating boilerplate code for methods like __init__, __repr__, and __eq__.

50. How does Python's type hinting improve code quality and readability?

Type hinting in Python allows developers to specify the expected data types of function arguments and return values. This improves code quality and readability by making the code more self-documenting and helping with static type checking.

from collections import Counter

def min_window(s, t):
    if not s or not t:
        return ""
    dict_t = Counter(t)
    required = len(dict_t)
    l, r = 0, 0
    formed = 0
    window_counts = {}
    ans = float("inf"), None, None
    while r < len(s):
        character = s[r]
        window_counts[character] = window_counts.get(character, 0) + 1
        if character in dict_t and window_counts[character] == dict_t[character]:
            formed += 1
        while l <= r and formed == required:
            character = s[l]
            if r - l + 1 < ans[0]:
                ans = (r - l + 1, l, r)
            window_counts[character] -= 1
            if character in dict_t and window_counts[character] < dict_t[character]:
                formed -= 1
            l += 1
        r += 1
    return "" if ans[0] == float("inf") else s[ans[1]: ans[2] + 1]

from collections import OrderedDict

class LRUCache:
    def __init__(self, capacity: int):
        self.cache = OrderedDict()
        self.capacity = capacity

    def get(self, key: int) -> int:
        if key not in self.cache:
            return -1
        else:
            self.cache.move_to_end(key)
            return self.cache[key]

    def put(self, key: int, value: int) -> None:
        if key in self.cache:
            self.cache.move_to_end(key)
        self.cache[key] = value
        if len(self.cache) > self.capacity:
            self.cache.popitem(last=False)

37. Implement a Circular Queue

Write a Python function to implement a circular queue.

class CircularQueue:
    def __init__(self, k: int):
        self.queue = [None] * k
        self.head = self.tail = -1
        self.size = k

    def enQueue(self, value: int) -> bool:
        if (self.tail + 1) % self.size == self.head:
            return False
        elif self.head == -1:
            self.head = 0
        self.tail = (self.tail + 1) % self.size
        self.queue[self.tail] = value
        return True

    def deQueue(self) -> bool:
        if self.head == -1:
            return False
        elif self.head == self.tail:
            self.head = self.tail = -1
        else:
            self.head = (self.head + 1) % self.size
        return True

    def Front(self) -> int:
        if self.head == -1:
            return -1
        return self.queue[self.head]

    def Rear(self) -> int:
        if self.tail == -1:
            return -1
        return self.queue[self.tail]

    def isEmpty(self) -> bool:
        return self.head == -1

    def isFull(self) -> bool:
        return (self.tail + 1) % self.size == self.head

def lowest_common_ancestor(root, p, q):
    if not root or root == p or root == q:
        return root

def diameter_of_binary_tree(root):
    def depth(node):
        nonlocal diameter
        if not node:
            return 0
        left = depth(node.left)
        right = depth(node.right)
        diameter = max(diameter, left + right)
        return max(left, right) + 1

    diameter = 0
    depth(root)
    return diameter

48. Implement a Hash Map

Write a Python function to implement a hash map.

class ListNode:
    def __init__(self, key=None, val=None):
        self.key = key
        self.val = val
        self.next = None

class HashMap:
    def __init__(self, size=1000):
        self.size = size
        self.table = [None] * size

    def put(self, key, value):
        index = hash(key) % self.size
        if not self.table[index]:
            self.table[index] = ListNode(key, value)
        else:
            current = self.table[index]
            while current:
                if current.key == key:
                    current.val = value
                    return
                if not current.next:
                    break
                current = current.next
            current.next = ListNode(key, value)

    def get(self, key):
        index = hash(key) % self.size
        current = self.table[index]
        while current:
            if current.key == key:
                return current.val
            current = current.next
        return -1

    def remove(self, key):
        index = hash(key) % self.size
        current = self.table[index]
        if not current:
            return
        if current.key == key:
            self.table[index] = current.next
        else:
            prev = current
            current = current.next
            while current:
                if current.key == key:
                    prev.next = current.next
                    return
                prev = current
                current = current.next

def max_profit(prices):
    min_price = float('inf')
    max_profit = 0
    for price in prices:
        if price < min_price:
            min_price = price
        elif price - min_price > max_profit:

    return max_profit
